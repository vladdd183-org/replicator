# Research — Комплексный ресерч

Комплексный ресерч, объединяющий поиск в кодовой базе, онлайн и документации.

## Когда использовать

- Перед реализацией новой фичи
- Когда нужно понять "как правильно" что-то сделать
- Для сбора информации по теме из всех источников

## Доступные режимы

| Команда | Режим | Описание |
|---------|-------|----------|
| `agent-os/research` | Комплексный | Все источники |
| `agent-os/research-codebase` | Код | Только существующий код |
| `agent-os/research-online` | Онлайн | Веб, документация, статьи |

## Процесс комплексного ресерча

### PHASE 1: Определить тему

IF пользователь указал тему, используй её.

OTHERWISE спроси:

```
Что вы хотите исследовать?

Укажите тему и я проведу комплексный ресерч:
- В существующем коде проекта
- В документации библиотек (локальной и онлайн)
- В статьях и best practices

Пример: "Как реализовать WebSocket handlers в Hyper-Porto"
```

### PHASE 2: Ресерч в кодовой базе

Сначала ищем существующие примеры в проекте:

1. **Структурный поиск** — найди похожие компоненты
2. **Паттерны** — как реализовано в проекте
3. **Стандарты** — проверь `agent-os/standards/`

### PHASE 3: Ресерч в официальной документации

Проверь официальные источники:

- Litestar — <https://docs.litestar.dev/>
- Piccolo ORM — <https://piccolo-orm.com/docs/>
- Dishka DI — <https://dishka.dev/>
- Returns — <https://returns.readthedocs.io/>
- Strawberry GraphQL — <https://strawberry.rocks/docs/>
- anyio — <https://anyio.readthedocs.io/>

### PHASE 4: Онлайн ресерч (при необходимости)

Если локальной информации недостаточно:

1. **Context7** — для документации библиотек
2. **Web search** — для статей и best practices
3. **arXiv** — для архитектурных паттернов (редко)

### PHASE 5: Синтез и рекомендации

Объедини информацию из всех источников:

```markdown
## 📚 Комплексный ресерч: [Тема]

### Из кодовой базы проекта

**Существующие примеры:**
- `путь/к/файлу.py` — [что делает, как реализовано]

**Используемые паттерны:**
- [Паттерн]: [как применяется]

### Из документации

**Официальные рекомендации:**
- [Библиотека]: [рекомендация]

**Примеры из GitHub:**
- `https://github.com/...` — [релевантный пример]

### Из онлайн-источников

**Best Practices:**
- [Practice]: [описание]

**Полезные статьи:**
- [Название] — [URL]

---

## 🎯 Итоговые рекомендации

### Как реализовать в Hyper-Porto

1. **Структура:** [какие файлы создать, где]
2. **Паттерн:** [какой паттерн следовать]
3. **Пример для копирования:** [конкретный файл]

### Пример кода

```python
# Пример реализации на основе ресерча
...
```

### Чеклист перед реализацией

- [ ] Изучить: `путь/к/примеру.py`
- [ ] Следовать стандарту: `agent-os/standards/...`
- [ ] Использовать шаблон: `agent-os/templates/...`
```

## Сохранение результатов ресерча

Если ресерч проводится для спецификации, сохрани результаты:

```
agent-os/specs/[текущая-спека]/
└── planning/
    └── research.md    # Результаты ресерча
```

## Быстрые команды

| Запрос | Что исследуем |
|--------|---------------|
| `research WebSocket` | WebSocket handlers |
| `research GraphQL mutations` | GraphQL мутации |
| `research Background tasks` | TaskIQ фоновые задачи |
| `research Caching` | Кэширование с cashews |
| `research Authentication` | JWT аутентификация |
| `research Testing Actions` | Тестирование Actions |
