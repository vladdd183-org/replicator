# Feedback — Обратная связь

Сбор, анализ и применение обратной связи для улучшения работы агентов.

## Когда использовать

- После завершения задачи — запросить оценку
- Для просмотра метрик качества работы
- Для анализа трендов и проблемных областей
- Для применения улучшений на основе фидбека

## Синтаксис

```
/agent-os/feedback [действие]
```

## Доступные действия

| Действие | Описание |
|----------|----------|
| `collect` | Запросить фидбек по последней задаче |
| `stats` | Показать текущие метрики |
| `dashboard` | Полный dashboard с трендами |
| `analyze` | Запустить анализ накопленного фидбека |
| `insights` | Показать извлечённые инсайты |
| `actions` | Список pending actions на основе фидбека |
| `approve {id}` | Одобрить предложенное действие |
| `reject {id}` | Отклонить действие |
| `survey` | Запустить еженедельный опрос |

## Примеры использования

### Запросить фидбек

```
/agent-os/feedback collect
```

Агент задаст краткие вопросы:
1. Оценка результата (1-5)
2. Соответствует ли требованиям?
3. Комментарий (опционально)

### Показать метрики

```
/agent-os/feedback stats
```

Выведет:
- Средняя оценка удовлетворённости
- Процент успешных задач
- Процент ошибок
- Среднее количество итераций

### Полный dashboard

```
/agent-os/feedback dashboard
```

Покажет:
- Текущие метрики с трендами
- Проблемные области
- Успешные области
- Pending actions

### Анализ фидбека

```
/agent-os/feedback analyze
```

Запустит анализ:
- Поиск повторяющихся проблем
- Обнаружение паттернов в негативном фидбеке
- Создание insights и предложений actions

### Инсайты

```
/agent-os/feedback insights
```

Покажет извлечённые наблюдения:
- Часто встречающиеся проблемы
- Области для улучшения
- Успешные паттерны

## Процесс

Делегируется агенту **feedback-collector**:

```
@.cursor/agents/feedback-collector.md

Выполни действие с фидбеком:

1. Определи тип действия
2. Используй структуру agent-os/feedback/
3. Обновляй метрики (metrics/current.json)
4. Создавай insights при обнаружении паттернов

Конфигурация: @agent-os/feedback/config.yml
```

## Структура системы

```
agent-os/feedback/
├── config.yml           # Конфигурация триггеров
├── collection/          # Сбор фидбека
│   ├── triggers.yml     # Когда спрашивать
│   ├── questions/       # Шаблоны вопросов
│   └── responses/       # Сырые ответы
├── analysis/            # Анализ
│   ├── rules.yml        # Правила анализа
│   ├── reports/         # Отчёты
│   └── insights/        # Инсайты
├── actions/             # Действия
│   ├── pending/         # Ожидающие
│   ├── applied/         # Применённые
│   └── rejected/        # Отклонённые
└── metrics/             # Метрики
    ├── current.json     # Текущие
    ├── history/         # История
    └── dashboards/      # Визуализации
```

## Автоматические триггеры

Фидбек автоматически запрашивается:
- После завершения реализации
- После исправления ошибки
- После сессии > 15 минут
- По пятницам (еженедельный опрос)

Можно отключить в `config.yml`.

## Типы вопросов

```
👍 или 👎?
```

```
1. Оценка (1-5):

```

### Detailed (для weekly survey)
Полный набор вопросов по разным аспектам.
