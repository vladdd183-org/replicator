# Research Codebase — Ресерч существующего кода

Анализ существующих паттернов, компонентов и примеров в кодовой базе проекта.

## Когда использовать

- Перед созданием нового компонента — найти похожие примеры
- Для понимания как реализованы существующие фичи
- Для переиспользования существующих паттернов

## Процесс

### PHASE 1: Определить что искать

IF пользователь указал что искать, используй это.

OTHERWISE спроси:

```
Что вы хотите найти в кодовой базе?

Примеры:
- "Как реализованы Actions" — найду примеры Actions
- "Паттерн работы с Repository" — найду Repository примеры
- "Примеры GraphQL resolvers" — найду GraphQL код
- "Как отправляются Events" — найду работу с Events
```

### PHASE 2: Поиск в кодовой базе

Используй следующую стратегию поиска:

1. **Поиск по структуре** — найди релевантные папки:
```bash
# Найти все Actions
find src/Containers -name "*Action.py" -type f

# Найти все Tasks  
find src/Containers -name "*Task.py" -type f

# Найти все Controllers
find src/Containers -path "*/UI/API/*" -name "*.py" -type f
```

2. **Поиск по паттернам** — найди конкретные реализации:
```bash
# Найти использование Result
grep -r "Result\[" src/ --include="*.py" -l

# Найти использование UnitOfWork
grep -r "async with self.uow" src/ --include="*.py" -l

# Найти публикацию Events
grep -r "add_event\|\.emit\(" src/ --include="*.py" -l
```

3. **Семантический поиск** — используй codebase_search для понимания:
   - "Как работает CreateUserAction?"
   - "Где используется Repository pattern?"
   - "Как реализована аутентификация?"

### PHASE 3: Анализ найденного

Для каждого найденного примера:

1. **Прочитай код** целиком
2. **Выдели паттерн** — как структурирован код
3. **Отметь зависимости** — что импортируется, что инжектится
4. **Найди особенности** — нестандартные решения

### PHASE 4: Сформируй отчёт

Выведи структурированный отчёт:

```markdown
## 🔍 Результаты ресерча: [Тема]

### Найденные примеры

**1. [Название файла]** (`путь/к/файлу.py`)
- Назначение: [что делает]
- Паттерн: [какой паттерн использует]
- Особенности: [что интересного]

**2. [Название файла]** (`путь/к/файлу.py`)
- ...

### Общие паттерны

- [Паттерн 1]: Используется в [где]
- [Паттерн 2]: Используется в [где]

### Рекомендации для новой реализации

- Переиспользовать: [что можно взять]
- Следовать паттерну: [какому]
- Избегать: [что не делать]

### Полезные файлы для копирования

- `путь/к/примеру.py` — хороший пример [чего]
```

## Быстрые команды

Для типичных запросов:

| Запрос | Что ищем |
|--------|----------|
| `research-codebase Actions` | Все примеры Actions |
| `research-codebase Repository` | Паттерны Repository |
| `research-codebase Events` | Работа с Domain Events |
| `research-codebase GraphQL` | GraphQL resolvers |
| `research-codebase Auth` | Аутентификация |
| `research-codebase Tests` | Примеры тестов |
