# Ask Orchestrator

Единая точка входа в AI-систему Hyper-Porto. Принимает любой запрос на русском или английском языке.

## Синтаксис

```
/agent-os/ask [любой запрос]
```

## Как это работает

Orchestrator автоматически:
1. Анализирует ваш запрос
2. Определяет намерение (разработка, исследование, отладка и т.д.)
3. Выбирает нужный агент или skill
4. Координирует выполнение
5. Возвращает результат

## Примеры использования

### Разработка

```
/agent-os/ask создай action для регистрации пользователя
/agent-os/ask добавь endpoint для получения профиля
/agent-os/ask сделай модуль для уведомлений
/agent-os/ask напиши тесты для UserModule
/agent-os/ask добавь graphql resolver для users
```

### Spec-Driven Development

```
/agent-os/ask новая фича — экспорт отчётов
/agent-os/ask реализуй спеку user-notifications
/agent-os/ask проверь реализацию текущей спеки
/agent-os/ask создай задачи по спеке
```

### Исследование и документация

```
/agent-os/ask как в litestar сделать middleware
/agent-os/ask документация по piccolo миграциям
/agent-os/ask найди где используется UserRepository
/agent-os/ask как работает Result pattern в проекте
/agent-os/ask лучшие практики для domain events
```

### Отладка

```
/agent-os/ask ошибка dishka — dependency not provided
/agent-os/ask не работает result — pattern matching падает
/agent-os/ask миграция не применяется
```

### Self-Improving

```
/agent-os/ask запомни: в этом проекте используем UUID v4
/agent-os/ask что я делал неправильно с импортами
/agent-os/ask вспомни решение по кешированию
/agent-os/ask обнови стандарты кодирования
```

### Обучение

```
/agent-os/ask научи меня Result pattern
/agent-os/ask как начать работу с проектом
/agent-os/ask туториал по Actions
```

## Контекст

Orchestrator автоматически учитывает:
- **Memory** — ваши предпочтения, предыдущие решения, ошибки
- **Standards** — стандарты кодирования проекта
- **Current Module** — если вы работаете в контексте модуля

## Мультишаговые задачи

Для сложных задач (например, "новая фича") Orchestrator запустит полный workflow:

```
/agent-os/ask новая фича — система уведомлений
```

Будет выполнено:
1. ✅ Инициализация спеки
2. ✅ Сбор требований (интерактивно)
3. ✅ Написание спецификации
4. ✅ Создание списка задач
5. ⏸️ [Ожидание подтверждения]
6. ⏸️ Реализация
7. ⏸️ Верификация

## Отличия от прямых команд

| Прямая команда | /agent-os/ask |
|----------------|---------------|
| `/dev/add-action` | `ask создай action` |
| `/agent-os/knowledge` | `ask как работает X` |
| `/agent-os/memory` | `ask запомни/вспомни` |
| `/agent-os/training` | `ask научи меня` |

**Преимущество /ask:**
- Не нужно помнить конкретные команды
- Автоматический выбор нужного инструмента
- Координация сложных задач
- Контекстная память
