## 🎯 Введение

**MAKER** (Maximal Agentic decomposition, first-to-ahead-by-K Error correction, and Red-flagging) — это революционная система, которая впервые в истории успешно решила задачу с **более чем миллионом шагов LLM без единой ошибки** 🎉

```mermaid
mindmap
  root((🧠 MAKER))
    🎯 Цель
      Миллион+ шагов
      Ноль ошибок
      Масштабируемость
    🔧 Методы
      Декомпозиция
      Голосование
      Фильтрация
    💡 Результат
      Надёжность
      Эффективность
      Контроль
```

### 🌟 Что делает MAKER особенным?

| Характеристика | Традиционные LLM | MAKER |
|----------------|------------------|-------|
| 🔢 Максимум шагов | ~100-300 | **1,000,000+** |
| ❌ Ошибки | Накапливаются | **Исправляются** |
| 📈 Масштабирование | Деградация | **Логарифмический рост** |
| 🎯 Фокус агента | Вся задача | **Один шаг** |

### 😱 Почему обычные LLM не справляются?

```mermaid

        A[🚀 Шаг 1] --> B[📝 Шаг 2]
        B --> C[📝 Шаг 3]
        C --> D[💥 Ошибка!]
        D --> E[📝 Шаг 5]

    end

    style G fill:#ff6b6b,stroke:#c92a2a,stroke-width:3px
```

**🔴 Главная проблема:** Даже при точности 99% на каждом шаге:
- После 100 шагов → ~37% шанс успеха
- После 1000 шагов → ~0.004% шанс успеха  
- После 1,000,000 шагов → **практически 0%** 😰

## ⚙️ Три Столпа Системы

### 🔨 Столп 1: Максимальная Декомпозиция (MAD)

> **Идея:** Разбить огромную задачу на мельчайшие возможные подзадачи, где каждый агент отвечает только за ОДИН шаг.

```mermaid
flowchart LR
    subgraph BEFORE["❌ БЕЗ декомпозиции"]
        BIG[🤖 ОДИН Агент<br/>1,000,000 шагов<br/>💥 ПЕРЕГРУЗКА]
    end
    
    subgraph AFTER["✅ С декомпозицией MAD"]
        MA1[🤖 1 шаг]
        MA2[🤖 1 шаг]
        MA3[🤖 1 шаг]
        MAN[🤖 1 шаг]
        MADOTS[...<br/>1M агентов]
    end
    
    BIG -.->|"Трансформация"| MA1
    
    style BIG fill:#ff6b6b,stroke:#c92a2a
    style MA1 fill:#69db7c,stroke:#2f9e44
    style MA2 fill:#69db7c,stroke:#2f9e44
    style MA3 fill:#69db7c,stroke:#2f9e44
    style MAN fill:#69db7c,stroke:#2f9e44
```

#### 🎯 Преимущества MAD:

```mermaid
mindmap
  root((🔨 MAD))
    🎯 Фокус
      Каждый агент знает только своё
      Минимальный контекст
      Нет путаницы
    🔄 Модульность
      Независимые шаги
      Возможность переделать
      Параллелизация
    💰 Эффективность
      Маленькие модели работают
      Меньше токенов
      Быстрее ответы
    🛡️ Надёжность
      Ошибка изолирована
      Легко исправить
      Голосование возможно
```

#### 📊 Как работает один микроагент:

```mermaid
sequenceDiagram
    participant S as 📍 Состояние
    participant T as 📝 Шаблон
    participant M as 🤖 LLM
    participant P as 🔍 Парсер
    participant N as 📍 Новое состояние

    S->>T: Текущее состояние задачи
    T->>M: Сформированный промпт
    Note over M: Агент думает<br/>только об ОДНОМ шаге
    M->>P: Сырой ответ
    P->>N: Действие + Новое состояние
    
    Note over S,N: 🔄 Процесс повторяется<br/>для следующего шага
```

### 🗳️ Столп 2: Голосование "First-to-ahead-by-K"

> **Идея:** Для каждого шага запускаем несколько независимых агентов и выбираем ответ, который "выиграл" с отрывом в K голосов.

```mermaid
flowchart TB
    subgraph STEP["📝 Один Шаг Задачи"]
        TASK_STEP[🎯 Подзадача]
    end
    
    subgraph SAMPLES["🎲 Независимые Выборки"]
        S1[🤖 Выборка 1]
        S2[🤖 Выборка 2]
        S3[🤖 Выборка 3]
        S4[🤖 Выборка 4]
        S5[🤖 Выборка 5]
        SN[🤖 ...]
    end
    
    subgraph VOTING["🗳️ Процесс Голосования"]
        COUNT[📊 Подсчёт голосов]
        CHECK{🔍 Есть ли<br/>отрыв в K?}
        MORE[➕ Нужно больше<br/>выборок]
        WINNER[🏆 Победитель!]
    end
    
    TASK_STEP --> S1 & S2 & S3 & S4 & S5 & SN
    S1 & S2 & S3 & S4 & S5 & SN --> COUNT
    COUNT --> CHECK
    CHECK -->|Нет| MORE
    MORE -->|Новая выборка| SAMPLES
    CHECK -->|Да| WINNER
    
    style WINNER fill:#69db7c,stroke:#2f9e44,stroke-width:3px
    style CHECK fill:#ffd43b,stroke:#fab005
```

#### 🎮 Пример голосования (K=3):

```mermaid
graph TB
    subgraph VOTES["🗳️ Голоса за разные ответы"]
        A["Ответ A: 🟢🟢🟢🟢🟢🟢<br/>(6 голосов)"]
        B["Ответ B: 🔴🔴🔴<br/>(3 голоса)"]
        C["Ответ C: 🟡<br/>(1 голос)"]
    end
    
    subgraph CALC["📊 Расчёт"]
        DIFF["Отрыв: 6 - 3 = 3<br/>K = 3"]
        RESULT["✅ Ответ A ПОБЕДИЛ!<br/>Отрыв ≥ K"]
    end
    
    A --> DIFF
    B --> DIFF
    DIFF --> RESULT
    
    style A fill:#69db7c
    style RESULT fill:#69db7c,stroke:#2f9e44,stroke-width:3px
```

#### 🔄 Алгоритм голосования:

```mermaid
stateDiagram-v2
    [*] --> Инициализация: Начало шага
    
    Инициализация --> Выборка: Запрос к LLM
    
    Выборка --> Валидация: Получен ответ
    
    Валидация --> RedFlag: Есть проблемы?
    Валидация --> Подсчёт: Ответ валиден
    
    RedFlag --> Выборка: Отбросить, повторить
    
    Подсчёт --> Проверка: Обновить счётчики
    
    Проверка --> Победа: Отрыв ≥ K
    Проверка --> Выборка: Отрыв < K
    
    Победа --> [*]: Вернуть результат
    
    note right of RedFlag: 🚩 Слишком длинный ответ?<br/>❌ Неверный формат?
    note right of Победа: 🏆 Уверенный ответ найден!
```

### 🚩 Столп 3: Red-Flagging (Красные Флаги)

> **Идея:** Если ответ LLM выглядит подозрительно — отбросить его, не пытаясь "починить".

```mermaid
flowchart TB
    subgraph INPUT["📥 Входящий Ответ"]
        RAW[🤖 Сырой ответ от LLM]
    end
    
    subgraph FLAGS["🚩 Проверка Красных Флагов"]
        F1{📏 Слишком<br/>длинный?}
        F2{❌ Неверный<br/>формат?}
        F3{🔄 Зациклился<br/>в рассуждениях?}
    end
    
    subgraph DECISION["🎯 Решение"]
        REJECT[🗑️ ОТБРОСИТЬ<br/>Запросить новый]
        ACCEPT[✅ ПРИНЯТЬ<br/>В голосование]
    end
    
    RAW --> F1
    F1 -->|Да >750 токенов| REJECT
    F1 -->|Нет| F2
    F2 -->|Да| REJECT
    F2 -->|Нет| F3
    F3 -->|Да| REJECT
    F3 -->|Нет| ACCEPT
    
    style REJECT fill:#ff6b6b,stroke:#c92a2a,stroke-width:2px
    style ACCEPT fill:#69db7c,stroke:#2f9e44,stroke-width:2px
```

## 🔄 Потоки Данных и Взаимодействия

### 🌊 Полный Поток Выполнения

```mermaid
sequenceDiagram
    autonumber
    participant T as 🎯 Задача
    participant D as 🔨 Декомпозитор
    participant A as 🤖 Агенты
    participant V as 🗳️ Голосование
    participant R as 🚩 Red-Flag
    participant S as 💾 Состояние
    participant O as 📤 Результат

    T->>D: Большая задача (1M шагов)
    D->>D: Разбить на подзадачи
    
    loop Для каждого шага
        D->>A: Подзадача i + Текущее состояние
        
        loop Пока нет победителя
            A->>R: Ответ агента
            
            alt Есть красный флаг
                R-->>A: 🚩 Отклонено, повторить
            else Ответ валиден
                R->>V: ✅ Добавить голос
            end
            
            V->>V: Проверить отрыв
            
            alt Отрыв >= K
                V->>S: 🏆 Победитель найден
            else Отрыв < K
                V-->>A: Нужно больше голосов
            end
        end
        
        S->>D: Обновлённое состояние
    end
    
    S->>O: ✅ Финальный результат
```

## 📊 Сравнение Подходов

### 🆚 Монолитный vs MAKER

## 📚 Глоссарий

| Термин | Описание |
|--------|----------|
| 🔨 **MAD** | Maximal Agentic Decomposition — максимальная декомпозиция на микроагентов |
| 🗳️ **First-to-ahead-by-K** | Правило голосования: побеждает тот, кто набрал на K голосов больше |
| 🚩 **Red-Flagging** | Отсев подозрительных ответов по косвенным признакам |
| 🤖 **Микроагент** | Агент, отвечающий за выполнение только одного шага |
| 📊 **Декорреляция** | Уменьшение связи между ошибками разных выборок |
| 💾 **Состояние** | Минимальная информация, передаваемая между шагами |
