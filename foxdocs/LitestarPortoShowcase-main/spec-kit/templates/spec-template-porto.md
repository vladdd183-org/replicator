## ⚡ Краткие рекомендации (специфичные для Porto)
- ✅ Сосредоточься на ТОМ, что нужно пользователям и ЗАЧЕМ
- ✅ Мысли в терминах Porto: Containers, Actions, Tasks, Models
- ❌ Избегай описания КАК реализовывать (никаких конкретных технических деталей)
- 👥 Пиши для бизнес-заинтересованных лиц, а не для разработчиков

### Руководство по сопоставлению с Porto
- **Пользовательские истории** → **Actions** (слой оркестрации)
- **Бизнес-операции** → **Tasks** (атомарные операции)
- **Сущности данных** → **Models** (Piccolo ORM)
- **Пользовательские интерфейсы** → **UI/API/CLI/Web**

### Требования к разделам
- **Обязательные разделы**: Должны быть заполнены для каждой функции
- **Опциональные разделы**: Включать только когда актуально для функции
- Если раздел не применим, удали его полностью (не оставляй как "Н/П")

### Для генерации ИИ
При создании этой спецификации из пользовательского запроса:
1. **Сначала определи размещение Container**: AppSection vs VendorSection
2. **Отметь все неоднозначности**: Используй [ТРЕБУЕТ УТОЧНЕНИЯ: конкретный вопрос]
3. **Думай о Porto компонентах**: Какие Actions/Tasks/Models нужны?
4. **Рассмотри переиспользование Ship**: Какие существующие Ship компоненты можно использовать?

### 🚨 КРИТИЧНО: Правила импортов для ИИ
**ПРИ ГЕНЕРАЦИИ КОДА ВСЕГДА ИСПОЛЬЗУЙ ТОЛЬКО АБСОЛЮТНЫЕ ИМПОРТЫ:**

✅ **ПРАВИЛЬНО:**
```python
from src.Ship.Parents import Action, Task, Model
from src.Containers.AppSection.User.Models import User
from src.Containers.AppSection.User.Data.Dto import UserCreateDTO
```

❌ **ЗАПРЕЩЕНО:**
```python
from ..Models import User          # Относительные импорты
from ....Actions import SomeAction # Глубокие относительные
from ..shortcuts import UserDTO    # Shortcuts
from .Models import User           # Локальные импорты
```

**Это НЕ ПОДЛЕЖИТ ОБСУЖДЕНИЮ - нейронки должны ВСЕГДА использовать абсолютные импорты начинающиеся с `src.`**
