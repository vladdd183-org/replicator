**Обязательные поля отмечены ***

**Подсказки:**
- Если задача размера `L` — разбейте на 2-3 подзадачи размера `S`/`M`.
- Критерии приёмки = контракт между постановщиком и исполнителем. Чем конкретнее, тем меньше переделок.
- Не пишите «Должно работать быстро» — пишите «Ответ API < 200ms на p95».
- Пустая задача без Acceptance Criteria не берётся в работу.
