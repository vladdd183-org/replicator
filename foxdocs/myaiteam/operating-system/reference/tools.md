## 3. Критерии выбора инструмента

Перед добавлением любого инструмента — пройди чек-лист:

- [ ] **Self-hosted или бесплатный?** (ограничение: zero budget)
- [ ] **Достаточно простой?** Команда реально будет пользоваться, а не «ещё один инструмент»
- [ ] **Интегрируется с текущим стеком?** API, webhooks, совместимость
- [ ] **Async-native?** Работает без синхронного присутствия всех участников
- [ ] **Searchable?** Можно найти информацию через 6 месяцев
- [ ] **Есть API для автоматизации?** Можно подключить AI-агентов

ЕСЛИ инструмент не проходит ≥ 4 из 6 пунктов — не добавляем.

---

## 4. Как предложить новый инструмент

1. Проверь критерии из раздела 3
2. Попробуй сам в течение 1 недели на реальной задаче
3. ЕСЛИ критерии пройдены — напиши короткий RFC:
   - Что за инструмент
   - Какую проблему решает
   - Чем лучше текущего решения
   - Результат личного тестирования
4. Команда пробует 2 недели
5. ЕСЛИ работает — добавляем в этот документ + обновляем MANIFEST

> **Антипаттерн:** «Давайте попробуем X, я видел на Хабре». Без личного тестирования — не обсуждаем.

## 5. Серверные мощности

| Сервер | CPU | RAM | Используется для |
|--------|-----|-----|-----------------|
| Сервер 1 | 16 потоков | 64 ГБ | Сервисы, CI/CD, Huly, Gitea |

> **Ограничение:** серверы используются для нескольких проектов. Перед деплоем нового сервиса — согласование с DevOps / Архитектором.

## 6. При масштабировании

Инструменты для рассмотрения на уровне 2 (20+ человек):

| Инструмент | Зачем | Когда рассматривать |
|-----------|-------|-------------------|
| Discord / Slack | Замена Telegram для командных каналов | Telegram-каналы становятся неуправляемыми |
| Loom | Async-видео вместо созвонов | Текстовые объяснения не справляются |
| Cal.com | Self-hosted планировщик встреч | > 5 внешних встреч в неделю |
| Figma | Дизайн-коллаборация | Появление выделенного дизайнера |

> **Правило:** не добавляем инструмент «на вырост». Только когда текущий стек не справляется с конкретной задачей.
