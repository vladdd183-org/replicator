# Глоссарий

**Версия:** 1.0  
**Дата:** 2026-02-09

---

## Цель документа

Быстрый справочник терминов, используемых в Operating System FatData Team. Открыл — нашёл — закрыл. Без воды.

Для полного IT-глоссария (аббревиатуры, жаргонизмы, инструменты, роли) → см. [Глоссарий FatData Team](../../../WorkRegulations/Документация/Глоссарий.md).

## Область применения

| Роль | Ответственность |
|------|----------------|
| Все участники | Используют единую терминологию |
| Автор документа | Добавляет новый термин при первом использовании |

## Процессы и методологии

| Термин | Определение |
|--------|------------|
| ADR | Architecture Decision Record — документ, фиксирующий принятое решение и его обоснование |
| DRI | Directly Responsible Individual — один конкретный человек, отвечающий за решение |
| DoR | Definition of Ready — чек-лист: можно ли начинать этап |
| DoD | Definition of Done — чек-лист: можно ли закрывать этап |
| RFC | Request for Comments — документ для async-обсуждения решения до его принятия |
| SLA | Service Level Agreement — договорённость о максимальном времени ответа |
| RACI | Responsible, Accountable, Consulted, Informed — матрица распределения ответственности |
| Async-first | Асинхронность по умолчанию. Синхронные встречи — исключение, не правило |
| Handbook | Живой справочник команды — единый источник правды по процессам |
| Playbook | Пошаговая инструкция для конкретной ситуации (онбординг, инцидент, релиз) |

## AI-коллаборация

### Роли AI в команде

| Термин | Определение |
|--------|------------|
| Generator | AI-роль: создаёт черновики документов, кода, предложений |
| Critic | AI-роль: анализирует риски, находит слабые места |
| Synthesizer | AI-роль: объединяет данные из разных источников |
| Scribe | AI-роль: автоматически обновляет документацию по итогам обсуждений |
| Verifier | AI-роль: проверяет метрики и KPI |

### Фреймворки и модели

| Термин | Определение |
|--------|------------|
| HIT | Hybrid Intelligence Team — фреймворк AI-коллаборации для команд |
| HITL | Human-in-the-Loop — человек участвует на каждом шаге, AI только предлагает |
| HOTL | Human-on-the-Loop — AI действует, человек наблюдает и вмешивается при необходимости |
| HOOTL | Human-out-of-the-Loop — полная автономия AI (у нас — только для рутинных задач) |
| Shadow Mode | AI автоматически мониторит обсуждения и обновляет документацию без запроса |
| Diamond Model | Малое ядро людей принимает решения + AI-агенты на периферии выполняют рутину |
| COMPASS | AI Strategy Compass — фреймворк для определения AI-стратегии команды |
| MAKER | Maximal Agentic Knowledge-Enhanced Reasoning — подход к мультиагентным системам |
