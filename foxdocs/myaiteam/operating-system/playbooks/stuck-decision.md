# Playbook: Не можем принять решение — что делать

## TL;DR

Чекни матрицу решений → Назначь DRI → Дай дедлайн → Не решили? Sync 20 мин → DRI решает → Disagree and commit.

## Когда использовать

- Обсуждение в чате идёт по кругу больше 24 часов.
- Два и более участника имеют противоположные мнения и не приходят к согласию.
- Задача блокирована из-за отсутствия решения.
- Никто не берёт ответственность за выбор.

## Шаги

### 1. Любой участник: обозначь тупик

- Напиши в чат: "Мы застряли на решении [тема]. Нужен DRI и дедлайн."
- ЕСЛИ тема есть в [матрице решений](../modules/decision-making/README.md) — посмотри, кто принимает это решение по умолчанию.

### 2. Менеджер (или инициатор): назначь DRI

- DRI (Directly Responsible Individual) — один человек, который примет финальное решение.
- Напиши в чат: "[Имя] — DRI по вопросу [тема]. Дедлайн решения: [дата, не позже 48ч]."
- ЕСЛИ непонятно кого назначить — DRI становится тот, кого решение затрагивает больше всего.

### 3. Команда: предоставьте аргументы DRI (до дедлайна)

- Каждый, у кого есть мнение, пишет в тред / задачу в Huly:
  - Вариант: [что предлагаю]
  - Плюсы: [1-3 пункта]
  - Минусы: [1-3 пункта]
- Формат короткий — не эссе, а факты.

### 4. ЕСЛИ дедлайн близко и консенсуса нет → sync на 20 минут

- DRI собирает голосовой звонок (Telegram).
- Формат: каждый участник — 3 минуты на свою позицию. Потом DRI задаёт вопросы.
- НЕ затягивай дольше 20 минут. Цель — услышать аргументы, не переубедить.

### 5. DRI: прими решение и зафиксируй

- Напиши в чат:

```
РЕШЕНИЕ по [тема]:
Выбрано: [вариант]
Причина: [1-2 предложения]
Действия: [что делаем дальше]
```

- Создай или обнови задачу в Huly с принятым решением.
- ЕСЛИ решение значимое (влияет на архитектуру / процессы) — оформи как [ADR](../modules/decision-making/adr-process.md).

### 6. Команда: disagree and commit

- Решение принято — выполняем, даже если не согласны.
- НЕ саботируй, НЕ возвращайся к обсуждению без новых фактов.
- ЕСЛИ через 2 недели решение оказалось ошибочным — любой может инициировать пересмотр с данными.

## Если что-то пошло не так

| Проблема | Решение |
|---|---|
| DRI не принимает решение в срок | Менеджер напоминает. ЕСЛИ снова молчит — менеджер принимает решение сам |
| Никто не хочет быть DRI | Менеджер назначает принудительно. Это обязанность, не опция |
| Решение вызвало конфликт | Sync 1-на-1 между менеджером и недовольным. Обсудить конструктивно |
| Новые факты опровергают решение | Инициируй пересмотр: покажи данные, назначь нового DRI |
