# Модуль: project-workflow

**Версия:** 1.0  
**Статус:** ✅ Active  
**DRI:** Старший менеджер

---

## Зачем этот модуль

Определяет **этапы проектной работы** и правила перехода между ними. Без него — хаос: непонятно, что считается «готовым», кто проверяет качество передачи, и какие этапы можно пропустить.

Модуль отвечает на три вопроса:
1. Какие этапы проходит задача от запроса клиента до production?
2. Что нужно сделать, чтобы перейти на следующий этап (DoR/DoD)?
3. Кто отвечает за каждый переход?

## Когда активен

Всегда. Это базовый процесс работы с задачами — действует для всех проектов FatData Team.

## Содержимое модуля

| Файл | Описание |
|------|----------|
| `stages.md` | Этапы, критические переходы (DoR/DoD), типы проектов, упрощённый режим |
| `templates/brief.md` | Шаблон брифа проекта |
| `templates/task.md` | Шаблон задачи (User Story + AC) |
| `templates/pr.md` | Шаблон Pull Request |
| `templates/adr.md` | Шаблон Architecture Decision Record |
| `templates/handover.md` | Шаблон Handover-пакета |

## Зависимости

| Модуль | Зачем нужен |
|--------|-------------|
| `core/roles-and-tiers` | Кто выполняет какую роль на каждом этапе |
| `core/decision-framework` | Как принимаются архитектурные решения на этапе проектирования (RFC/ADR) |
| `core/communication` | SLA на ответы, формат сообщений при передаче между этапами |

## Критерии отключения

Модуль нельзя отключить — он базовый. Можно адаптировать: добавить этапы, упростить чек-листы, изменить ответственных. Любые изменения — через PR с ревью от DRI модуля.
