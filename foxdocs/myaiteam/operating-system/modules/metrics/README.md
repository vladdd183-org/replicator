# Модуль: Metrics

**Версия:** —  
**Статус:** 📋 Planned  
**DRI:** Тимлид

---

## Назначение

Минимальный набор метрик для оценки здоровья процессов. Не для микроменеджмента — для раннего обнаружения проблем.

## Триггер

Ядро процессов стабильно **2+ месяца** + проведена хотя бы одна ретро.

## Содержимое модуля

| Файл | Описание |
|------|----------|
| `health-metrics.md` | Базовые (Level 1), расширенные (Level 2) метрики, Team Health Survey |

## Зависимости

| Модуль | Зачем |
|--------|-------|
| `core/communication` | Время ответа привязано к SLA |
| `decision-making/adr` | Метрика «% решений с ADR» |
| `project-workflow` | Возвраты задач привязаны к DoR/DoD |

## Критерии отключения

Отключаем через 2 месяца после активации, если:
- Ни одна метрика не привела к конкретному действию
- Сбор данных стоит больше, чем даёт пользы
- Команда воспринимает метрики как контроль
