# Модуль: Team Scaling

**Версия:** —  
**Статус:** 📋 Planned  
**DRI:** CEO / Архитектор  
**Триггер активации:** 20+ человек в команде

---

## Назначение

Модуль определяет **как команда FatData масштабируется** — когда делиться на подкоманды, как выстроить взаимодействие между ними и какие процессы включать по мере роста. Сейчас нас 14, и одна плоская команда работает. Этот модуль — план на будущее. Активируется по триггерам, не по расписанию.

## Содержимое модуля

| Файл | Описание |
|------|----------|
| `README.md` | Этот файл — обзор модуля |
| `team-topologies.md` | План разделения на подкоманды по модели Team Topologies + Diamond Model |
| `scaling-triggers.md` | Явные сигналы: когда активировать новые модули и процессы |

## Зависимости

| Модуль | Зачем |
|--------|-------|
| `core/roles-and-tiers.md` | Новые роли (Team Lead, Tech Lead) строятся поверх текущей ролевой модели |
| `core/decision-framework.md` | RFC/DRI-процесс масштабируется на кросс-командные решения |
| `core/communication.md` | Протоколы связи адаптируются: team-каналы, cross-team sync |
| `decision-making/rfc` | RFC становится основным инструментом согласования между командами |
| `ai-collaboration` | Diamond Model (Level 3) напрямую связана с AI-стратегией |

## Критерии отключения

Модуль отключается, ЕСЛИ команда сократилась до <15 человек и мульти-командная структура избыточна. При повторном росте — реактивация.
