## Назначение

Модуль определяет **self-service онбординг** — процесс, в котором новый участник вводится в команду самостоятельно, опираясь на документацию и чек-листы. Buddy помогает с вопросами, но **не является источником знаний** — источник знаний — это доки.

Философия: **«Всё есть в документации. Buddy — для вопросов, а не для передачи знаний.»**

ЕСЛИ новичок не нашёл ответ в документации — это баг в документации, а не повод для созвона.

## Зависимости

| Модуль | Зачем |
|--------|-------|
| `core/principles.md` | Новичок читает в первую очередь — фундамент всех решений |
| `core/communication.md` | Каналы, SLA, формат сообщений — без этого не получится работать |
| `core/roles-and-tiers.md` | Роли и уровни участия — понять, куда ты попал |
| `core/decision-framework.md` | Как принимаются решения — чтобы не блокировать и не быть заблокированным |

> **Важно:** новый участник ОБЯЗАН прочитать все документы из `core/` в первую неделю. Без этого остальная документация не имеет контекста.

## Критерии отключения

Модуль отключается, если команда перестаёт расти и онбординг не нужен. При возобновлении найма — реактивация.
