# Модуль: Decision Making

**Версия:** 1.0  
**Дата:** 2026-02-09  
**DRI:** Тимлид / Архитектор

---

## Назначение

Модуль формализует **как команда принимает и фиксирует решения**. Два инструмента:

- **ADR** (Architecture Decision Records) — фиксация решений **после** их принятия. Неизменяемый лог: почему выбрали именно так.
- **RFC** (Request for Comments) — обсуждение решений **до** их принятия. Async-дискуссия с дедлайном и DRI.

RFC предлагает варианты → команда обсуждает → DRI решает → ADR фиксирует результат.

## Зависимости

- **`core/decision-framework.md`** — матрица решений, DRI-принцип, уровни делегирования. Этот модуль **реализует** процессы, описанные во фреймворке.
- **`core/communication.md`** — каналы, SLA, формат сообщений. RFC и ADR используют эти каналы.

## Критерии отключения RFC

RFC — эксперимент. Отключаем, если по итогам апреля 2026:
- Ни один RFC не был создан за 2 месяца (процесс не нужен)
- RFC регулярно игнорируются (команда не готова)
- Время принятия решений **увеличилось** из-за бюрократии
