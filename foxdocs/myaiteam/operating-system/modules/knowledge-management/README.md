## Назначение

Модуль определяет **как команда хранит, структурирует и находит знания**. Цель — собрать всё в единый handbook и выстроить культуру Handbook-First: если чего-то нет в документации — этого не существует.

Два направления:
- **Handbook-First** (ближайшее) — структура справочника, Docs-as-Code процесс
- **Knowledge Graph** (долгосрочное) — эволюция к AI-powered графу знаний на базе исследований `myaiteam/`

## Критерии активации

Модуль переходит в 🧪 Experiment, когда:
1. Команда определилась с хранилищем (Gitea repos / отдельный handbook-repo)
2. Есть хотя бы 3 документа, которые реально используются участниками
3. DRI модуля назначен и готов поддерживать handbook
