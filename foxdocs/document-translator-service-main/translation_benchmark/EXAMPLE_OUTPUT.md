## 📊 Интерпретация результатов

### Скорость (Время на 1 текст)
- **M2M100: 0.378s** - быстрее за счёт прямого перевода
- **Argos: 0.512s** - медленнее из-за двухэтапного перевода (zh→en→ru)

### Использование CPU
- **M2M100: 75.8%** - активно использует все ядра процессора
- **Argos: 41.3%** - менее интенсивные вычисления

### Использование памяти
- **M2M100: ~1GB** - большая модель требует больше RAM
- **Argos: ~325MB** - компактные модели, меньше памяти

### Качество перевода
Обе модели дают хорошие результаты, но:
- **M2M100** может быть точнее за счёт прямого перевода
- **Argos** может терять нюансы при двойном переводе

## 🎯 Выводы для выбора

### Выбирайте Argos Translate если:
- ✅ Ограничена оперативная память (< 4GB)
- ✅ Нужен лёгкий переводчик
- ✅ Не критична максимальная скорость
- ✅ Подходит хорошее качество (не идеальное)

### Выбирайте M2M100 если:
- ✅ Достаточно памяти (> 4GB)
- ✅ Важна максимальная скорость
- ✅ Критична точность перевода
- ✅ Есть мощный многоядерный процессор

## 💡 Примечания

### Первый запуск
При первом запуске M2M100 будет загружать модель (~500MB):
```
🔧 Инициализация M2M100 (m2m100_418M)...
  📥 Загрузка модели (может занять время при первом запуске)...
Downloading (…)lve/main/config.json: 100%|████████| 1.42k/1.42k [00:00<00:00, 891kB/s]
Downloading model.safetensors: 100%|████████████████| 493M/493M [02:15<00:00, 3.63MB/s]
Downloading (…)okenizer_config.json: 100%|████████| 42.0/42.0 [00:00<00:00, 28.1kB/s]

```

Последующие запуски будут использовать кешированную модель.

### Вариативность результатов
Метрики могут варьироваться в зависимости от:
- 🖥️ Мощности процессора
- 🧠 Доступной памяти
- ⚙️ Загруженности системы
- 📝 Длины и сложности текстов

### Точность измерений
- Время измеряется с точностью до миллисекунд
- CPU% показывает использование относительно одного ядра (может быть > 100%)
- RAM показывает резидентную память процесса (RSS)
