### Файл: `pyi_rth_paddlex.py` (НОВЫЙ)

Runtime hook который патчит проверку зависимостей PaddleX:

```python
# Импортирует paddlex.utils.deps сразу при запуске
import paddlex.utils.deps

# Патчит функцию require_extra чтобы она ничего не делала
def patched_require_extra(extra, obj_name=None, alt=None):
    # Пропускаем проверку - все зависимости уже в EXE
    return

paddlex.utils.deps.require_extra = patched_require_extra
```

Это позволяет PaddleX работать без файлов метаданных пакетов.

## 🐛 Если что-то пошло не так

### Проблема: Всё ещё множественные процессы

**Решение:**
1. Убедитесь что пересобрали EXE после изменений
2. Проверьте что установлена переменная: `$env:APP_ENV = "production"`
3. Удалите старую сборку и пересоберите заново:
   ```powershell
   Remove-Item -Recurse -Force dist, build
   .\build-windows.ps1
   ```

### Проблема: Всё ещё ошибка DependencyError

**Решение:**
1. Убедитесь что файл `DocumentTranslator.spec.template` содержит все изменения
2. Проверьте что spec файл копируется правильно при сборке
3. Пересоберите с нуля:
   ```powershell
   Remove-Item -Recurse -Force venv, dist, build
   .\build-windows.ps1
   ```

### Проблема: Долгая инициализация OCR

Это нормально при первом запуске - модели скачиваются. Подождите 5-10 минут.
Последующие запуски будут быстрее.

## 🔍 Технические детали

### Почему это работает?

1. **is_frozen()** определяет запуск из PyInstaller через `sys.frozen` и `sys._MEIPASS`
2. **copy_metadata()** включает файлы `.dist-info` с информацией о версиях пакетов
3. **collect_all('Cython')** включает все utility файлы Cython включая `.cpp` файлы
4. **Дополнительные импорты** гарантируют что все модули PaddleX присутствуют
5. **Runtime hook `pyi_rth_paddlex.py`** патчит проверку зависимостей ДО запуска приложения
   - Hook выполняется первым, до импорта вашего кода
   - Импортирует `paddlex.utils.deps` и заменяет функцию `require_extra`
   - Новая функция просто возвращается без проверки метаданных

### Почему PyInstaller не включал это автоматически?

- PyInstaller анализирует импорты статически
- Метаданные пакетов не являются Python модулями
- Cython utility файлы загружаются динамически
- PaddleX импортирует модули по строковым именам
