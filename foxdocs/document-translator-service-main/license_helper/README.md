# 🔐 License Helper Service

Сервис для проверки лицензий Document Translator на стороне клиента.

## 📋 Что это?

`license-helper` - это небольшой сервис, который:
1. Получает уникальный ID вашей машины (Machine ID)
2. Проверяет лицензию на нашем сервере
3. Предоставляет информацию Docker контейнеру

**Поддерживаемые ОС:** Windows, Linux

## 🚀 Установка

### Windows

#### Вариант 1: Запуск вручную (для тестирования)

1. Скачайте `license-helper.exe`
2. Запустите двойным кликом
3. Готово! Сервис работает

#### Вариант 2: Установка как Windows Service (рекомендуется)

1. Запустите `install_as_service.bat` **от имени администратора**
2. Сервис автоматически запустится и будет стартовать при загрузке Windows

Удаление сервиса:
```batch
uninstall_service.bat
```

### Linux

#### Вариант 1: Запуск напрямую (для тестирования)

```bash
# Установка зависимостей и запуск
./run.sh
```

#### Вариант 2: Сборка бинарника

```bash
# Сборка
./build.sh

# Запуск собранного бинарника
./dist/license-helper
```

#### Вариант 3: Установка как systemd service (рекомендуется)

```bash
# Сначала соберите бинарник
./build.sh

# Затем установите как сервис
sudo ./install_as_service.sh

# Проверка статуса
sudo systemctl status license-helper

# Просмотр логов
sudo journalctl -u license-helper -f
```

Удаление сервиса:
```bash
sudo ./uninstall_service.sh
```

**Примечание для Linux:**
- На некоторых системах может потребоваться `sudo` для получения Machine ID
- Для получения hardware UUID используется `dmidecode` или `/sys/class/dmi/id/product_uuid`
- Если нет прав root, будет использован MAC-адрес как fallback

## 🔍 Как получить Machine ID?

### Способ 1: Через браузер

1. Запустите `license-helper.exe`
2. Откройте браузер: http://localhost:9999/machine-id
3. Скопируйте Machine ID

### Способ 2: Через PowerShell

```powershell
Invoke-RestMethod -Uri http://localhost:9999/machine-id
```

### Способ 3: Через curl

```bash
curl http://localhost:9999/machine-id
```

## 🐛 Решение проблем

### "Не могу запустить .exe"

**Проблема:** Windows блокирует запуск

**Решение:**
1. Правый клик на файл → Свойства
2. Внизу: "Разблокировать" → Применить
3. Или добавьте папку в исключения Windows Defender

### "Cannot get machine ID"

**Проблема:** Нет прав доступа к WMI

**Решение:**
1. Запустите от имени администратора
2. Или установите как службу (install_as_service.bat)

### Сервис не запускается после перезагрузки

**Проблема:** Служба отключена

**Решение:**
1. Откройте "Службы" (services.msc)
2. Найдите "DocTransLicenseHelper"
3. Тип запуска → "Автоматически"
4. Запустить

## 🔐 Безопасность

### Можно ли подделать Machine ID?

❌ Нет. Machine ID генерируется из:
- UUID материнской платы
- Серийного номера диска

### Можно ли использовать одну лицензию на нескольких машинах?

❌ Нет. Лицензия привязана к Machine ID конкретной машины.

## 📝 FAQ

### Q: Нужен ли интернет для работы?

**A:** Да, сервис проверяет лицензию онлайн при каждом запросе (с кешированием).

### Q: Как часто проверяется лицензия?

**A:** При каждом запросе к Document Translator, но результат кешируется на 5 минут.

### Q: Можно ли работать offline?

**A:** Нет, в текущей версии требуется постоянное подключение к интернету.

### Q: Что будет при истечении лицензии?

**A:** Document Translator перестанет обрабатывать документы. Helper service будет показывать сообщение о необходимости продления.

### Q: Можно ли продлить лицензию заранее?

**A:** Да, дни добавятся к текущему сроку действия.

## 📊 Системные требования

### Windows
- **ОС:** Windows 10/11 (64-bit)
- **RAM:** 50 MB
- **Disk:** 20 MB
- **Порт:** 9999 (должен быть свободен)
- **Интернет:** Требуется для проверки лицензии
- **Права:** Администратор (для получения Machine ID через WMIC)

### Linux
- **ОС:** Ubuntu 20.04+, Debian 11+, или совместимые дистрибутивы
- **Python:** 3.8+ (для запуска напрямую)
- **RAM:** 50 MB
- **Disk:** 20 MB
- **Порт:** 9999 (должен быть свободен)
- **Интернет:** Требуется для проверки лицензии
- **Права:** root/sudo (для получения hardware UUID, опционально)
- **Утилиты:** `dmidecode` или `lsblk` (опционально, для hardware ID)
