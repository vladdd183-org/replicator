### Обновления CI/CD

#### Автоматические релизы
Теперь релизы создаются **автоматически** при каждом коммите в `main`:

- ⚡ **Триггер**: Push в ветку `main`
- 📁 **Условие**: Изменения в папке `license_helper/`
- 🏷️ **Версионирование**: Автоматическое (`vYYYY.MM.DD-SHA`)
- 🚀 **Артефакты**: Автоматически публикуются в GitHub Releases

#### Пример версии
```
v2024.10.22-a1b2c3d
  ↑      ↑       ↑
  |      |       └── Короткий SHA коммита
  |      └────────── Дата сборки
  └───────────────── Префикс версии
```

### Как работает новый workflow

1. **Вы коммитите изменения в `license_helper/`**
   ```bash
   git add license_helper/
   git commit -m "feat: добавил новую функцию"
   git push origin main
   ```

2. **GitHub Actions автоматически:**
   - Обфусцирует код с PyArmor
   - Собирает Windows EXE
   - Генерирует версию из даты + SHA
   - Создает релиз с бинарником

3. **Готово!** Релиз доступен в: `https://github.com/YOUR_REPO/releases`

### Преимущества

- ✅ Не нужно вручную создавать теги
- ✅ Не нужно придумывать номера версий
- ✅ Каждое изменение в `main` = новый релиз
- ✅ Полная история сборок с привязкой к коммитам
- ✅ Workflow запускается только при изменениях в `license_helper/`

### Файлы в папке

```
license_helper/
├── build-with-pyarmor.sh    # Локальная сборка с защитой
├── build.sh                  # Локальная сборка без защиты
├── install_as_service.sh     # Установка как systemd service
├── license_helper.py         # Основной код
├── README.md                 # Документация
├── requirements.txt          # Зависимости Python
├── run.sh                    # Запуск для разработки
└── uninstall_service.sh      # Удаление service
```

### Миграция из старой версии

Если у вас были теги вида `v1.0.0`, они продолжат работать, но новые релизы будут создаваться автоматически при push в `main`.

### Откат к ручным релизам

Если нужно вернуться к ручному созданию релизов через теги:

1. Откройте `.github/workflows/build-license-helper.yml`
2. Замените:
   ```yaml
   on:
     push:
       branches:
         - main
       paths:
         - 'license_helper/**'
   ```
   На:
   ```yaml
   on:
     push:
       tags:
         - 'v*.*.*'
   ```
