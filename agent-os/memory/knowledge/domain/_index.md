## 📝 Структура документации модуля

Каждый модуль может иметь:

```
domain/{Section}/{Module}/
├── overview.md      # Обзор: назначение, границы, API
├── decisions.md     # Локальные архитектурные решения
└── gotchas.md       # Подводные камни и особенности
```
