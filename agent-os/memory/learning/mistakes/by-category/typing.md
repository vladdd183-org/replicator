# Ошибки категории: Типизация

## Описание

Здесь собраны типичные ошибки связанные с типизацией Python: аннотации типов, дженерики, Protocol, TypeVar и совместимость типов.

## Частые ошибки

### 1. Отсутствие аннотаций типов

**Симптом:** Код работает, но IDE не подсказывает типы, mypy выдает ошибки.
**Причина:** Функции и переменные не аннотированы типами.
**Решение:** Всегда указывать типы параметров и возвращаемых значений.

```python
# ❌ Плохо
def process(data):
    return data.get("value")

# ✅ Хорошо
def process(data: dict[str, Any]) -> str | None:
    return data.get("value")
```

### 2. Использование Optional вместо X | None

**Симптом:** Устаревший синтаксис Optional[X].
**Причина:** Использование старого стиля типизации.
**Решение:** Использовать современный синтаксис X | None (Python 3.10+).

```python
# ❌ Устаревшее
from typing import Optional
def get_user(id: int) -> Optional[User]:
    ...

# ✅ Современное
def get_user(id: int) -> User | None:
    ...
```

### 3. Неправильное использование TypeVar

**Симптом:** Generic не работает как ожидалось, типы не выводятся.
**Причина:** TypeVar определен без bound или неправильно использован.
**Решение:** Правильно определять TypeVar с bound при необходимости.

```python
# ❌ Плохо — T может быть чем угодно
T = TypeVar("T")

# ✅ Хорошо — T ограничен BaseModel
T = TypeVar("T", bound=BaseModel)
```

### 4. Игнорирование ковариантности/контравариантности

**Симптом:** Ошибки типизации при наследовании с дженериками.
**Причина:** Неправильное понимание вариантности типов.
**Решение:** Использовать covariant=True для read-only, contravariant=True для write-only.
