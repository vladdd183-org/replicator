# Ошибки уровня: Critical (Критические)

## Описание

**Critical** ошибки — это ошибки которые **полностью блокируют работу приложения** или приводят к потере данных, нарушению безопасности, или невозможности запуска системы.

## Критерии

Ошибка относится к этому уровню если:
- Приложение не запускается или падает при старте
- Происходит потеря или повреждение данных
- Нарушается безопасность (утечка секретов, SQL-инъекции)
- Транзакции не откатываются при ошибках
- DI-контейнер не может разрешить критические зависимости
- Миграции ломают продакшн базу данных

## Примеры

### 1. Отсутствие регистрации Provider в контейнере

**Категория:** di
**Симптом:** `NoFactoryError: Cannot find factory for <ClassName>`
**Влияние:** Приложение падает при первом запросе к эндпоинту
**Решение:** Добавить provider в `Providers.py` модуля и зарегистрировать в `create_dishka_container()`

---

### 2. Неправильный scope в Dishka Provider

**Категория:** di
**Симптом:** `ScopeViolationError` или `RuntimeError: Session is closed`
**Влияние:** Запросы падают случайным образом, утечка ресурсов
**Решение:** Использовать `Scope.REQUEST` для Repository, UoW; `Scope.APP` для stateless Tasks

---

### 3. Коммит без UnitOfWork context manager

**Категория:** result-pattern
**Симптом:** Данные не сохраняются или транзакция не откатывается
**Влияние:** Потеря данных, неконсистентное состояние БД
**Решение:** Всегда использовать `async with self.uow:` перед `await self.uow.commit()`

---

### 4. Циклические импорты между модулями

**Категория:** imports
**Симптом:** `ImportError: cannot import name 'X' from partially initialized module`
**Влияние:** Приложение не запускается
**Решение:** Использовать Events для межмодульной коммуникации вместо прямых импортов

---

### 5. Секреты в коде или git

**Категория:** security
**Симптом:** API ключи, пароли в исходниках или .env закоммичен
**Влияние:** Компрометация системы, утечка данных
**Решение:** Использовать `.env` + `.gitignore`, проверять pre-commit hooks

---

### 6. Отсутствие await для async операций

**Категория:** async
**Симптом:** `RuntimeWarning: coroutine was never awaited` или данные не записываются
**Влияние:** Операции не выполняются, данные теряются
**Решение:** Добавить `await` перед всеми async вызовами

---

### 7. Использование pip вместо uv

**Категория:** tooling
**Симптом:** `ModuleNotFoundError` при запуске через `uv run`
**Влияние:** Зависимости не синхронизированы, проект не запускается
**Решение:** Использовать только `uv add`, `uv sync`, `uv run`

---
