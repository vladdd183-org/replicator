# Ошибки уровня: Minor (Мелкие)

## Описание

**Minor** ошибки — это ошибки которые **не влияют на функциональность**, но ухудшают качество кода, читаемость или нарушают стилевые соглашения.

## Критерии

Ошибка относится к этому уровню если:
- Код работает корректно, но нарушает code style
- Неоптимальные, но работающие решения
- Отсутствие docstrings или комментариев
- Неидиоматичный Python
- Избыточный или дублирующийся код
- Некорректное именование (но понятное)

## Примеры

### 1. Неправильное именование Action

**Категория:** naming
**Симптом:** `UserAction` вместо `CreateUserAction`
**Влияние:** Неясно что делает Action без чтения кода
**Решение:** Использовать паттерн `{Verb}{Noun}Action`

---

### 2. Отсутствие docstring в Action

**Категория:** documentation
**Симптом:** Нет описания Use Case
**Влияние:** Сложнее понять назначение без анализа кода
**Решение:** Добавить docstring с описанием бизнес-логики

---

### 3. Избыточные импорты

**Категория:** imports
**Симптом:** Импортированные модули не используются
**Влияние:** Замедление загрузки, визуальный шум
**Решение:** Запустить `uv run ruff check . --fix`

---

### 4. Hardcoded строки вместо констант

**Категория:** code-quality
**Симптом:** `"USER_NOT_FOUND"` в нескольких местах
**Влияние:** Риск опечаток, сложнее рефакторить
**Решение:** Вынести в константы или Enum

---

### 5. Длинные функции (> 30 строк)

**Категория:** code-quality
**Симптом:** Action.run() содержит много логики
**Влияние:** Сложнее тестировать и понимать
**Решение:** Выносить логику в Tasks, использовать extract method

---

### 6. print() вместо logging

**Категория:** logging
**Симптом:** `print(f"Debug: {value}")`
**Влияние:** Вывод не структурирован, нет уровней
**Решение:** Использовать `logfire.info()` или structlog

---

### 7. Неконсистентные кавычки

**Категория:** style
**Симптом:** Смешанное использование `'` и `"`
**Влияние:** Визуальная неконсистентность
**Решение:** Запустить `uv run ruff format .`

---

### 8. TODO без ticket reference

**Категория:** documentation
**Симптом:** `# TODO: fix later`
**Влияние:** TODO теряются и не отслеживаются
**Решение:** `# TODO(JIRA-123): описание задачи`

---

### 9. Неиспользуемые переменные

**Категория:** code-quality
**Симптом:** Переменная объявлена, но не используется
**Влияние:** Мёртвый код, путает читающего
**Решение:** Удалить или использовать `_` для игнорируемых значений

---
