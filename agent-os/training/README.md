# Interactive Training Mode

> Система обучения и онбординга для новых разработчиков Hyper-Porto

---

## Обзор

Interactive Training Mode — это интерактивная система обучения, которая помогает разработчикам освоить архитектуру Hyper-Porto через практику. Вместо чтения документации, вы сразу пишете код с поддержкой AI-агентов.

### Что включено

| Раздел | Описание | Для кого |
|--------|----------|----------|
| [Tutorials](./tutorials/) | Пошаговые туториалы с объяснениями | Новички |
| [Simulations](./simulations/) | Интерактивные сценарии-симуляции | Все уровни |
| [Sandbox](./sandbox/) | Песочница для экспериментов | Все уровни |

### Уровни сложности

- **Beginner** — Первое знакомство с архитектурой
- **Intermediate** — Углублённое изучение паттернов
- **Advanced** — Сложные сценарии и оптимизация

## Быстрый старт

### 1. Начать обучение

Скажи AI-агенту:

```
@training начать обучение
```

или

```
Хочу пройти обучение Hyper-Porto, начни с основ
```

### 2. Выбрать туториал

```
@training туториал 01-first-action
```

### 3. Запустить симуляцию

```
@training симуляция create-user-module
```

## Туториалы

Пошаговые руководства с подробными объяснениями.

### Доступные туториалы

| # | Название | Уровень | Время |
|---|----------|---------|-------|
| 01 | [Создание первого Action](./tutorials/01-first-action.md) | Beginner | 30 мин |
| 02 | [Spec-Driven Workflow](./tutorials/02-spec-driven-workflow.md) | Intermediate | 45 мин |
| 03 | [Отладка с агентами](./tutorials/03-debugging-with-agents.md) | Intermediate | 40 мин |

### Как проходить туториал

1. **Открой туториал** — прочитай введение и цели
2. **Следуй шагам** — выполняй инструкции последовательно
3. **Пиши код** — AI поможет проверить результат
4. **Задавай вопросы** — если что-то непонятно, спрашивай

```
@training проверь мой код
```

## Симуляции

Интерактивные сценарии, имитирующие реальные задачи.

### Как работают симуляции

1. **Сценарий** — описание задачи и контекста
2. **Цель** — что нужно достичь
3. **Шаги** — рекомендуемый путь решения
4. **Подсказки** — помощь, если застрял
5. **Проверка** — автоматическая валидация результата

### Доступные симуляции

| Симуляция | Уровень | Описание |
|-----------|---------|----------|
| [create-user-module](./simulations/create-user-module/) | Beginner | Создание модуля User с нуля |

### Команды для симуляций

```bash
# Начать симуляцию
@training симуляция create-user-module

# Получить подсказку
@training подсказка

# Проверить прогресс
@training статус

# Сбросить симуляцию
@training сброс
```

## Sandbox (Песочница)

Безопасное место для экспериментов. Создавай, ломай, учись!

### Использование

```
@training sandbox создай тестовый Action
```

Sandbox находится в `agent-os/training/sandbox/` — все изменения там не влияют на основной код.

Подробнее: [Sandbox README](./sandbox/README.md)

## Команды Training Mode

### Общие команды

| Команда | Описание |
|---------|----------|
| `@training старт` | Начать обучение с определения уровня |
| `@training статус` | Текущий прогресс обучения |
| `@training помощь` | Показать доступные команды |

### Туториалы

| Команда | Описание |
|---------|----------|
| `@training туториалы` | Список всех туториалов |
| `@training туториал <name>` | Начать конкретный туториал |
| `@training проверь` | Проверить выполнение текущего шага |

### Симуляции

| Команда | Описание |
|---------|----------|
| `@training симуляции` | Список всех симуляций |
| `@training симуляция <name>` | Начать симуляцию |
| `@training подсказка` | Получить подсказку |
| `@training решение` | Показать решение (только для обучения!) |

### Sandbox

| Команда | Описание |
|---------|----------|
| `@training sandbox` | Войти в песочницу |
| `@training sandbox очистить` | Очистить песочницу |

## Прогресс обучения

Система отслеживает ваш прогресс:

```yaml
# Пример прогресса (хранится в config.yml)
progress:
  tutorials:
    01-first-action: completed
    02-spec-driven-workflow: in_progress
    03-debugging-with-agents: not_started
  simulations:
    create-user-module: completed
  level: intermediate
  total_time: "2h 15m"
```

### Просмотр прогресса

```
@training мой прогресс
```

## FAQ

### Я застрял, что делать?

```
@training подсказка
```

Агент даст подсказку без готового решения.

### Как проверить, что я всё сделал правильно?

```
@training проверь
```

Агент проверит код и укажет на ошибки.

### Можно ли пропустить туториал?

Да, но рекомендуется проходить последовательно. Каждый туториал опирается на предыдущие знания.

### Где хранится мой код?

- Туториалы: изменения в основном проекте (можно откатить через git)
- Симуляции: временные файлы (удаляются после)
- Sandbox: `agent-os/training/sandbox/`

## Добавление своих материалов

### Новый туториал

1. Создай файл в `tutorials/` с номером
2. Следуй шаблону существующих туториалов
3. Добавь в `tutorials/_index.md`

### Новая симуляция

1. Создай папку в `simulations/`
2. Добавь `scenario.yml`, `steps.md`, `hints.md`
3. Добавь в `simulations/_index.md`
