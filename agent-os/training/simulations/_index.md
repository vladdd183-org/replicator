# Симуляции Hyper-Porto

> Интерактивные сценарии для практики реальных задач

---

## Обзор

Симуляции — это практические задания, имитирующие реальные задачи разработки. В отличие от туториалов, здесь меньше подсказок и больше самостоятельности.

### Как работают симуляции

1. **Сценарий** — описание задачи и контекста
2. **Цель** — что нужно достичь
3. **Критерии успеха** — как понять, что готово
4. **Шаги** — рекомендуемый путь (можешь идти своим)
5. **Подсказки** — помощь, если застрял
6. **Проверка** — автоматическая валидация

## Как проходить симуляцию

### 1. Начало

```
@training симуляция create-user-module
```

### 2. Чтение сценария

Агент покажет:
- Контекст задачи
- Что нужно сделать
- Критерии успеха

### 3. Работа

Пиши код! Используй команды:

```bash
# Проверить прогресс
@training статус

# Получить подсказку
@training подсказка

# Следующий шаг
@training шаг
```

### 4. Проверка

```
@training проверь
```

Агент проверит:
- Структуру файлов
- Код на соответствие архитектуре
- Тесты (если требуются)

### 5. Завершение

При успешной проверке симуляция завершается.

---

## Уровни подсказок

| Уровень | Что получишь |
|---------|--------------|
| 1 | Направление (куда смотреть) |
| 2 | Конкретика (что именно сделать) |
| 3 | Пример (фрагмент кода) |
| 4 | Решение (полный код) |

По умолчанию подсказки выдаются последовательно. Каждая следующая более конкретная.

## Оценка

После завершения симуляции ты получишь оценку:

```yaml
score: 85/100
breakdown:
  structure: 20/20      # Правильная структура файлов
  code_quality: 18/20   # Качество кода
  architecture: 20/25   # Соответствие архитектуре
  tests: 15/20          # Наличие и качество тестов
  time: 12/15           # Бонус за время

hints_used: 2
time_spent: "38 минут"
```

### Критерии оценки

- **Structure (20)** — правильная структура папок и файлов
- **Code Quality (20)** — чистый код, нет lint ошибок
- **Architecture (25)** — соответствие Hyper-Porto
- **Tests (20)** — покрытие тестами
- **Time (15)** — бонус за быстрое выполнение

## Создание своей симуляции

### Структура

```
simulations/
└── my-simulation/
    ├── scenario.yml    # Описание сценария
    ├── steps.md        # Пошаговые инструкции
    ├── hints.md        # Подсказки по уровням
    └── validation.py   # Скрипт валидации (опционально)
```

### scenario.yml

```yaml
name: "Название симуляции"
level: beginner | intermediate | advanced
estimated_time: "45 мин"

description: |
  Описание задачи

goal: |
  Что нужно достичь

success_criteria:
  - Критерий 1
  - Критерий 2

prerequisites:
  - tutorial:01-first-action
```

### steps.md

```markdown
# Шаги симуляции

## Шаг 1: Название
Описание...

## Шаг 2: Название
Описание...
```

### hints.md

```markdown
# Подсказки

## Шаг 1

### Уровень 1
Направление...

### Уровень 2
Конкретика...
```
